Every async/await function is syntactic sugar over Promises. Here's what the engine actually does:

async/await (sugar)

async function getData() {
  const a = await fetchA();
  const b = await fetchB(a);
  return b;
}

Promise equivalent (desugared)

function getData() {
  return fetchA()
    .then(a => {
      return fetchB(a);
    })
    .then(b => {
      return b;
    });
}

Key Insight

await does three things: (1) Wraps the expression in Promise.resolve(), (2) Pauses execution of the async function, (3) Schedules the rest of the function as a microtask (like .then()). The calling code continues synchronously.

How async/await Works

  • async function always returns a Promise
  • await expr wraps expr in Promise.resolve() and suspends the function
  • Code after await runs as a microtask when the promise resolves
  • Synchronous code outside the async function runs first

Execution Order Rules

  • Synchronous code runs to completion first
  • Microtasks (Promise callbacks, await continuations) drain between each macrotask
  • Multiple awaits = multiple microtask checkpoints
  • await of an already-resolved promise still yields to the microtask queue

Common Gotchas

  • await in a loop = sequential execution. Use Promise.all() for parallel.
  • Forgetting await = fire-and-forget. The promise runs but errors are swallowed.
  • return await is usually unnecessary (except inside try/catch).
  • Top-level await only works in ES modules, not scripts.

Promise States

  • Pending: Initial state, not yet settled
  • Fulfilled: Operation completed successfully
  • Rejected: Operation failed with an error
  • Once settled, a promise never changes state again
